routing set the latlang on road but poliline just connect them directly so if a gps device is on a circular road its better to use route

First Look :

First you need mother code :

//header <script src="https://ajax.googleapis.com/ajax/libs/jquery/3.3.1/jquery.min.js"></script> <script src="root/js/leaflet.js"></script> <link rel="stylesheet" type="text/css" href="root/css/leaflet.css"> <script src="root/js/leaflet-routing-machine.js"></script> <link rel="stylesheet" type="text/css" href="root/css/leaflet-routing-machine.css"> //body <center><div id="map" style="width: 400px;height: 400px"></div></center> //end of the body <script> var map = L.map('map',{ center: [46.687888,7.872871], zoom: 15 }); L.tileLayer('http://{s}.tile.osm.org/{z}/{x}/{y}.png', { attribution: '&copy; <a href="http://osm.org/copyright">OpenStreetMap</a> contributors', maxZoom: 18, }).addTo(map); </script> To creating the route you need some location
L.Routing.control({ waypoints: [ L.latLng(46.686004, 7.869654), L.latLng(46.687888,7.872871), L.latLng(46.687888,7.872871) ], addWaypoints : false , draggableWaypoints : false , routeWhileDragging: false , createMarker: function() { return null; } }).addTo(map); if we dont add addWaypoints and draggableWaypoints and routeWhileDragging , user will be able to change the route by dragging
if we dont add createMarker: function() { return null; } they will need icon in root/css/images/icon-marker that we dont have so we return null
for removing the box that shows in top to describe the route you can use this style :
.leaflet-touch .leaflet-bar { display :none !important }
